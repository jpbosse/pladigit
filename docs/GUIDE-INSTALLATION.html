Guide d'installation

Ce guide vous accompagne pas-à-pas dans l'installation de Pladigit sur votre serveur. Aucune connaissance technique avancée n'est requise.

Ubuntu 24.04 LTS Version 1.0.0 Avril 2026
Pladigit — AGPL-3.0 — pladigit.fr — github.com/jpbosse/pladigit

Introduction

Pladigit est une plateforme de digitalisation open source conçue pour les collectivités locales françaises. Elle remplace les outils Microsoft (Teams, SharePoint, OneDrive, Office) par des alternatives souveraines, hébergées sur votre propre serveur.

Ce guide vous explique comment installer Pladigit en deux grandes étapes :

1
Le script d'installation prépare le serveur
Une seule commande installe automatiquement PHP, MySQL, Redis, Nginx et le code de Pladigit. Durée : 5 à 10 minutes.
2
L'assistant web configure Pladigit
Depuis votre navigateur, vous renseignez les paramètres (base de données, adresse du site, mot de passe administrateur). Durée : 5 minutes.

Système d'exploitation requis

Ubuntu Server 24.04 LTS — téléchargeable gratuitement sur ubuntu.com/download/server

ℹ️ Ubuntu Server vs Desktop : choisissez la version Server (sans interface graphique). Elle est plus légère et mieux adaptée à un serveur.

Chapitre 1 — Installation automatique
Le script installe PHP, MySQL, Redis, Nginx et Pladigit

Connexion au serveur

Connectez-vous à votre serveur via SSH depuis votre ordinateur. Ouvrez un terminal (sous Windows : PowerShell ou PuTTY ; sous Mac/Linux : Terminal) et tapez :

Terminal — Connexion SSH
ssh ubuntu@ADRESSE-IP-DU-SERVEUR

Remplacez ADRESSE-IP-DU-SERVEUR par l'adresse IP de votre serveur (ex: 192.168.1.10 ou 51.210.35.12).

Lancer l'installation

Une fois connecté à votre serveur, copiez-collez cette commande et appuyez sur Entrée :

Commande d'installation — À copier-coller
curl -fsSL https://pladigit.fr/get-install | sudo bash
⚠️ Mot de passe : après avoir appuyé sur Entrée, le système peut vous demander votre mot de passe. Tapez-le (les caractères ne s'affichent pas) et appuyez sur Entrée.

Suivi de l'installation

Le script affiche sa progression en temps réel. Vous verrez défiler les 7 étapes de l'installation. C'est normal si cela prend plusieurs minutes, notamment lors de la mise à jour du système et de l'installation des dépendances.

✅ Installation réussie — quand vous voyez "Environnement prêt !" et une URL commençant par http://, l'installation du serveur est terminée. Notez l'URL affichée.
🌐
Chapitre 2 — Assistant de configuration
8 étapes pour configurer Pladigit depuis votre navigateur

Depuis votre ordinateur (pas le serveur), ouvrez votre navigateur web et tapez l'URL affichée par le script :

URL à ouvrir dans votre navigateur
http://ADRESSE-IP-DU-SERVEUR/install/

Étape 1 — Bienvenue

La page d'accueil de l'assistant vous présente les 8 étapes de la configuration. Cliquez sur "Commencer l'installation".

Étape 2 — Vérification du système

L'assistant vérifie que votre serveur est compatible avec Pladigit. Toutes les lignes doivent afficher ✅. Si c'est le cas, cliquez sur "Continuer".

ℹ️ Si une ligne affiche ❌, c'est que le script d'installation a rencontré un problème. Consultez le fichier /var/log/pladigit-install.log sur votre serveur pour plus d'informations.

Étape 3 — Base de données

Cette étape configure la connexion à MySQL. Vous devez renseigner deux sections :

Section 1 — Connexion administrateur MySQL (root)

Laissez les valeurs par défaut (127.0.0.1, port 3306, utilisateur root). Le mot de passe root est vide sur une nouvelle installation Ubuntu — laissez le champ vide.

Section 2 — Compte dédié Pladigit

Choisissez un nom d'utilisateur (ex: pladigit) et définissez un mot de passe solide d'au moins 8 caractères. Ce compte sera utilisé par Pladigit pour accéder à ses données.

💡 Mot de passe root vide : sur Ubuntu, MySQL est installé sans mot de passe root par défaut. Laissez simplement le champ vide et cliquez sur "Tester et continuer".

Étape 4 — Paramètres de l'application

Renseignez l'adresse web de votre Pladigit et le nom de votre organisation.

URL : si vous n'avez pas encore de nom de domaine, utilisez l'adresse IP de votre serveur (ex: http://192.168.1.10). Vous pourrez configurer un domaine plus tard.

Étape 5 — Configuration email

Cette étape permet de configurer l'envoi d'emails (pour les invitations et réinitialisations de mot de passe). Elle est optionnelle — vous pouvez cliquer sur "Passer cette étape" et la configurer plus tard.

Étape 6 — Collabora Online

Collabora permet d'éditer des documents bureautiques (ODT, ODS, DOCX, XLSX) directement dans le navigateur, sans installer de logiciel sur les postes.

L'assistant détecte automatiquement l'espace disque disponible et vous recommande l'option la plus adaptée.

OptionDescriptionQuand choisir
Installer sur ce serveurCollabora installé automatiquement via Docker≥ 4 Go d'espace libre disponible
Utiliser une instance existanteRenseigner l'URL d'un serveur Collabora existantVous disposez déjà d'un serveur Collabora
PasserCollabora non configuré pour l'instantEspace insuffisant ou installation ultérieure

Étape 7 — Compte Super Administrateur

Créez le compte qui permettra d'administrer toute la plateforme : créer des organisations, gérer les utilisateurs, etc.

⚠️ Mot de passe : choisissez un mot de passe d'au minimum 12 caractères, difficile à deviner. Exemple : Mairie-2026-Pladigit!
Notez-le précieusement — vous en aurez besoin pour vous connecter.

Étape 8 — Lancement de l'installation

Toutes les informations sont saisies. Cliquez sur "Lancer l'installation" pour démarrer la configuration automatique.

Une barre de progression s'affiche avec le détail de chaque opération en cours (création de la base de données, migrations, cache...). L'opération prend généralement moins d'une minute.

Chapitre 4 — Premières étapes après installation
Créer votre première organisation et inviter vos agents

Créer votre première organisation

Depuis le tableau de bord Super Admin, cliquez sur "+ Nouvelle organisation" et renseignez :

1
Nom de l'organisation
Le nom de votre collectivité (ex: Mairie de Saint-Aubin-les-Communes)
2
Slug (identifiant)
Un identifiant court sans espace ni accent (ex: mairie-saint-aubin). Il sera utilisé dans l'URL de votre organisation.
3
Plan
Deux plans disponibles : Communautaire (gratuit, auto-géré) ou Partenaire (accompagnement sur devis). Choisissez "Communautaire" si vous gérez l'installation vous-même.

Accéder à votre organisation

Une fois l'organisation créée, connectez-vous en tant qu'administrateur de cette organisation via l'URL de votre serveur. Vous pourrez alors :

✓
Créer la structure organisationnelle
Directions, services, agents — reproduisez l'organigramme de votre collectivité.
✓
Inviter vos agents
Envoyez des invitations par email à chaque agent avec leur rôle défini.
✓
Créer vos premiers projets
Kanban, Gantt, gestion des jalons et des tâches.

Solutions aux problèmes courants

Le script s'arrête avec une erreur

Consultez le journal d'installation :

Afficher le journal d'installation
sudo tail -50 /var/log/pladigit-install.log

Espace disque insuffisant

Si votre serveur utilise LVM (Ubuntu Server par défaut), étendez le volume :

Étendre le volume LVM
sudo lvextend -l +100%FREE /dev/mapper/ubuntu--vg-ubuntu--lv && sudo resize2fs /dev/mapper/ubuntu--vg-ubuntu--lv
ℹ️ Le script d'installation détecte et corrige automatiquement ce problème depuis la version 1.0.0.

Erreur de connexion MySQL

Si le wizard indique "Access denied for user root", exécutez cette commande sur le serveur :

Corriger l'authentification MySQL
sudo mysql -e "ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY ''; FLUSH PRIVILEGES;"

Page d'erreur 500 après installation

Régénérez le cache de configuration :

Régénérer le cache
sudo -u www-data php /var/www/pladigit/artisan config:clear && sudo -u www-data php /var/www/pladigit/artisan config:cache
